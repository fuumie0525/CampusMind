# 校园网络故障排查规范

## 排查顺序

1. 确认影响范围：单设备、单账号、单楼层还是区域性故障。
2. 确认网络类型：校园 WiFi、有线网络、运营商宽带或学校 VPN。
3. 检查账号状态、认证页面、系统时间、代理和 DNS 配置。
4. 执行低风险操作：重新连接、忘记网络、切换网络、重新认证。
5. 记录错误码、发生时间、楼栋区域、设备系统和客户端版本。
6. 完成自查仍失败时，建议联系网络中心并提交必要诊断信息。

## 安全边界

- 不要求用户发送密码、验证码或完整身份证号。
- 不编造网络中心状态、账号封禁原因或后台日志。
- 不建议关闭系统安全防护、删除重要配置或无限重试。
- 区域性大面积断网、账号锁定或持续错误码应升级人工处理。
